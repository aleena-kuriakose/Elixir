Introduction to Elixir

What is Elixir?

Elixir is a dynamic general-purpose programming language that is designed to build scalable concurrent applications. Elixir allows developers to be productive in multiple domains including website development, embedded software, data pipelines, and multimedia processing. It was created in 2012 by Jose Valim, as a research and development project. It runs on top of the BEAM virtual machine, which was created in the 1980s, to build massively scalable soft real-time systems with high availability. Valim wanted to create something that could enable higher extensibility and productivity in the Erlang VM while keeping its compatibility. Elixir was aimed to handle large volumes of data by keeping a low latency. It adapts the features from Erlang, Ruby, and Clojure.


What is Erlang or OTP?

Erlang is a general purpose concurrent, functional programming language and garbage collecting concurrent system developed in 1986 by Ericsson for improved performance in telecommunication. The term Erlang and OTP(Open Telecom Platform) is interchangeably used, which consist of runtime system, several ready to use components written in Erlang program. Erlang was designed for systems with distributed, fault- tolerant and highly available traits. Talking about the processes, Erlang applications are built of very lightweight Erlang processes in Erlang runtime system . OTP (Open Telecom Platform) is a large collection of libraries for Erlang to d everything from compiling ASN1 to providing a WWW server. Whatsapp is coded using Erlang.


What makes Elixir different?

Scalability

Elixir code runs under lightweight threads of execution (processes) that exchanges the information via messages. Elixir has built-in schedular which makes it possible to scale vertically by adding more power to the machine. Due to concurrency, it is possible to scale horizontally by spawning multiple process in different machines in a cluster. Isolation is possible in elixir, which allows process to be garbage collected independently, reducing system wide pauses and resource usage, which helps in better efficiency. Processes can communicate with other processes in the same network, running in different machines.


Concurrency

Since elixir is built on top of erlang machine, it is possible to create multiple concurrent processes without any issues. An application running on Erlang virtual machine can make use of a lot of processes to execute functions in parallel as much as possible in a way more scalable than dealing with a common process. This is achieved by the actor model used by Erlang.

The Model: The actor model is used in Erlang and consist simply of actors yours messages sent to your mailboxes. The message is not sent to the actor indeed, but to its mailbox and periodically the actor checks the box to attend the next message on the queue. In short actors are single threaded processes which can send and receive message amongst themselves. The Erlang machine manages their creation, execution, and communication.


Fault-Tolerance

Erlang and Elixir have reputation for their fault tolerance. It is not because they prevent errors instead, they have tools to recover from the errors that interrupts the runtime. Unlike Java, Elixir uses supervisor behaviour to monitor the child processes, which means if a child process crashes, supervisor will restart it. It is more like when one user/ developer crashes, everything they used will be cleaned without affecting the performance of any other user. Supervisor modules spawn supervisor processes that link to other processes and watch for failures in the linked processes if they crash. This separation of concerns makes our code clearer and easier to maintain.

Functional Programming

Functional programming generally is using functions to the best effect for creating more understandable, clean, and maintainable code. Even though when Elixir is functional programming language, it is different from most of the other functional programming languages like Haskell or F#. This is because Elixir pragmatically handles concurrent systems with high fault tolerance. This means that Elixir is a functional programming language because this naturally fits it and not for its own sake. Since it runs on the top of the Erlang machine, it is inherently concurrent and distributed, but the syntax is very straight forward, unlike any other functional languages.

Dynamic Typing

Elixir is dynamically typed, just like Erlang and the BEAM. This means that types are checked in run-time, not during compilation. This can be a downside when building critical systems but when we think of creating simple web applications, it increases development speed. Moreover, static types can be introduced in Elixir through typespecs because they can be useful as they can be served as documentation and can also be used by static analysis tools like Dialyzer to find possible bugs..


Tooling

Elixir has a variety of development tools, which can make using elixir easier. Tooling is one of the best features of Elixir. Moreover, we can access and use the Erlang libraries, which has a mature ecosystem for decades. It includes Elixir’s interactive shell IEx, Mix, ExUnit, Credo, Dialyzer, TypedEchtoSchema, Kaffy, Libcluster.


IEx

IEx is your best friend when you are using Elixir because it does help a lot more than the usual windows PowerShell. IEx has the autocomplete functionality that helps in complete the unfamiliar new keywords or modules. Every time IEx starts , it starts looking for a .iex.exs file, if not present in the current directory, it searches in the home directory to find one. ‘h’ is one of the most useful tools in Elixir since it helps in finding the words in the code. It works somewhat like a search button and searches for that word. There are many similar prompts that helps in Elixir shell, for different purposes like i, r and t.


Credo

Credo is linter and static analysis linter tool that helps in improving elixir code quality. It helps in finding proper coding standards like using proper naming standards, improper or inconsistent language construct etc. Integrating credo into pre-commit helps in finding errors or warnings in the code.


Dialyzer

Since Elixir used dynamic typing, dialyzer is a tool that helps in using the static typing method in Elixir. Its more like dialyzer helps in finding all the errors and bugs the code would have if it was using static typing. Dialyzer helps in finding the bugs and also as a bonus, it discovers hidden bugs or other issues too.


Mix

Mix is used to compile and run your application, manage your dependencies, run tests and also is used for profiling the code. It helps in generating the projects folder structure and necessary boilerplate. Mix compiles the changes in the program when necessary, but it still will be needed to compile the project.


Kaffy

Kaffy is used in generating admin interface automatically from the Ecto scheme in the app. It is similar phpMyAdmin for PH[ or Active admins for Rails. The application might out grow Kaffy but to get an admin interface up and running quickly for internal usage, Kaffy is amazing.
